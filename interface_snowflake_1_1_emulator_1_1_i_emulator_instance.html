Detailed Description

While the IEmulatorAdapter is responsible for spawning an instance, the emulator instance does the heavy work. It is responsible for serializing valid configuration, and managing the state of the emulator process. While not all emulators will support certain states, it must at the minimal implement Create, Start, and Destroy. as well as clean up after itself in the event of an unexpected occurance.
